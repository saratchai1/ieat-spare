# อะไหล่แหลมฉบัง

เดโมเว็บแอปสำหรับระบบซ่อมบำรุงดิจิทัลทวินและคลังอะไหล่กลางของนิคมอุตสาหกรรมแหลมฉบัง

แอปนี้แสดงแนวคิดที่ผู้บริหารนิคมสามารถบริหารอะไหล่กลาง ทีมช่าง ใบแจ้งซ่อม การจองอะไหล่ และการแจ้งเตือนจากเซนเซอร์ของโรงงานผู้เช่าภายในนิคมเดียวกันได้จากศูนย์กลาง

## เทคโนโลยี

- Next.js พร้อม TypeScript
- Tailwind CSS
- Prisma ORM
- SQLite สำหรับฐานข้อมูลเดโม
- Recharts
- lucide-react สำหรับไอคอน

## วิธีติดตั้งและรัน

```bash
npm install
npm run db:push
npm run db:seed
npm run dev
```

เปิดที่ [http://localhost:3000](http://localhost:3000)

## ข้อมูลเดโม

สคริปต์ seed สร้างข้อมูลตัวอย่างดังนี้:

- โรงงานผู้เช่า 5 โรงงานในนิคมแหลมฉบัง
- เครื่องจักร 30 รายการ
- อะไหล่กลาง 20 รายการ
- ช่างซ่อมบำรุง 10 คน
- ใบแจ้งซ่อม 25 ใบ
- ค่าเซนเซอร์ 100 รายการ
- การแจ้งเตือนเชิงคาดการณ์
- รายการเคลื่อนไหวสต็อกและการจองอะไหล่

## Flow สำหรับนำเสนอ

1. เปิดศูนย์สั่งการและดูสัญญาณเตือนแรงสั่นสะเทือนวิกฤตของโรงงาน B
2. เปิดผังดิจิทัลทวินแบบ 2D site plan และเลือกโรงงาน B
3. เปิดมอเตอร์ที่ได้รับผลกระทบเพื่อดูแรงสั่น อุณหภูมิ กระแสไฟฟ้า ชั่วโมงเดินเครื่อง คะแนนสุขภาพ ประวัติซ่อม และอะไหล่ที่ใช้ร่วมกันได้
4. เปิดใบแจ้งซ่อมที่เกี่ยวข้อง
5. จองตลับลูกปืน 6308 จากคลังอะไหล่กลาง
6. มอบหมายช่างจากหน้าจัดทีมช่าง
7. อัปเดตสถานะงานและปิดงานพร้อมบันทึกอะไหล่ที่ใช้จริง
8. กลับมาดูศูนย์สั่งการเพื่อเห็นการเปลี่ยนแปลงของสถานะใบงาน สต็อก และ KPI

## บทบาทเดโม

ใช้ตัวสลับบทบาทที่แถบด้านบนหรือ sidebar:

- ผู้ดูแลนิคม
- ผู้ใช้งานโรงงาน
- ช่างซ่อมบำรุง
- เจ้าหน้าที่คลัง

ตัวสลับบทบาทเป็น mock สำหรับการนำเสนอเท่านั้น ไม่มีระบบยืนยันตัวตนจริง และไม่ต้องใช้บริการภายนอกแบบเสียเงิน

## หน้าหลักในระบบ

- `/` - ศูนย์สั่งการผู้บริหาร
- `/twin-map` - ผังดิจิทัลทวินแบบ 2D site plan ของนิคมแหลมฉบัง
- `/assets` - ทะเบียนเครื่องจักร
- `/inventory` - คลังอะไหล่กลาง
- `/tickets` - ระบบใบแจ้งซ่อม
- `/dispatch` - กระดานจัดทีมช่าง
- `/alerts` - การแจ้งเตือนเชิงคาดการณ์
- `/analytics` - มูลค่าทางธุรกิจและ KPI

## Deploy บน Vercel

โปรเจกต์นี้ใช้ SQLite สำหรับเดโม จึง bundle ไฟล์ `prisma/dev.db` ไปกับ repo และเมื่อรันบน Vercel แอปจะ copy ฐานข้อมูลไปไว้ที่ `/tmp/ieat-spare-demo.db` ก่อนเปิด Prisma Client เพื่อให้ serverless function อ่านและเขียนข้อมูลเดโมได้ใน instance นั้น

ตั้งค่า Vercel ได้ตามค่า default:

- Build Command: `npm run build`
- Install Command: `npm install`
- Output Directory: ปล่อยว่าง
- Environment Variables: ไม่ต้องตั้ง `DATABASE_URL` สำหรับเดโมนี้

ถ้าเคย deploy แล้วขึ้น `This page couldn't load` ให้ push commit ล่าสุดขึ้น GitHub แล้วกด Redeploy บน Vercel
